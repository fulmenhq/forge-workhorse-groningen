# Release 0.1.6 (2025-12-17)

Patch release focused on reducing CDRL friction by removing template residue from code and docs.

## Overview

- **CDRL hardening**: Eliminates template-name defaults and brittle test assertions that make refits fail.
- **Template neutrality**: `make help`, SBOM output, and `/version` fallbacks avoid hardcoding the template binary name.
- **Docs refresh**: Updates the Crucible-docs access guide to match current gofulmen/Crucible/toolchain versions.

## Changes

### Dependencies

- **gofulmen**: Upgraded to v0.1.22 (Crucible v0.2.23 transitively).

### CDRL Hardening

- CLI root defaults are template-neutral and overwritten by `.fulmen/app.yaml`.
- `/version` handler fallback uses the executable name rather than a hardcoded template string.
- Tests no longer assert template-specific identity values.

### Tooling

- Makefile help banner and SBOM output use `$(BINARY_NAME)`.

### Developer Docs

- Updated `docs/development/accessing-crucible-docs-via-gofulmen.md` wording to be template-neutral and aligned with current versions.

### Release Workflow

- Release workflow uploads `bin/*` rather than `bin/groningen-*`.
